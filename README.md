# allow_domains_custom

Персональный репозиторий со списками доменов для точечной маршрутизации трафика.  
Списки используются совместно с клиентами на базе sing-box / V2Ray.

---

## Версия sing-box

Файлы формата `.srs` скомпилированы под **sing-box 1.12.22**.

Версия обусловлена минимальной версией, используемой на клиентских устройствах, и требованием совместимости:
бинарный формат `.srs` обратно совместим (более новые клиенты читают файлы старых версий),
но не прямо совместим (файл собранный новой версией может не читаться старым клиентом).

---

## Структура репозитория

```
allow_domains_custom/
├── txt/        # Исходные списки доменов
├── json/       # Автогенерация: sing-box source rule-set
├── lst/        # Автогенерация: простой текстовый список доменов
└── srs/        # Автогенерация: скомпилированный бинарный rule-set
```

> Файлы в `json/`, `lst/` и `srs/` генерируются автоматически из исходников в `txt/`.

---

## Форматы файлов

### `txt/` — исходники

Простой текстовый файл, по одному домену на строку (только домены или только подсети).
Строки начинающиеся с `#` и пустые строки игнорируются при сборке.

```
# Комментарий
youtube.com
googlevideo.com
```

---

### `lst/` — текстовый список

Аналог `txt/`, но без комментариев — только домены или только подсети.

```
youtube.com
googlevideo.com
```
---

### `json/` — sing-box source rule-set

JSON-файл в формате sing-box Rule Set Source Format.  
Версия формата: **3** (поддерживается sing-box 1.11.0+).

Для доменов
```json
{
  "version": 3,
  "rules": [
    {
      "domain_suffix": [
        "youtube.com",
        "googlevideo.com"
      ]
    }
  ]
}
```
Для подсетей
```json
{
  "version": 3,
  "rules": [
    {
      "ip_cidr": [
        "11.222.0.0/11"
      ]
    }
  ]
}
```

---

### `srs/` — sing-box binary rule-set

Скомпилированный бинарный файл.
